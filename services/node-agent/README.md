# node-agent

Привилегированный агент узла (ADR-0041, спецификация docs/specs/adminchat-agentic-devops.md,
раздел 2). Это крошечный HTTP-сервер на стандартной библиотеке Python без внешних зависимостей,
который разворачивается DaemonSet-ом по одному поду на каждом узле кластера и даёт панели
adminchat возможность исполнять произвольные команды прямо на хосте узла.

## Назначение

Панель adminchat превращена в агентного девопса, который управляет не только приложением через
Kubernetes API, но и самими узлами. Значительная часть операций эксплуатации живёт мимо
Kubernetes API: чистка диска, prune образов и слоёв docker и containerd, наблюдение через df, du,
top, free, uptime, per-core загрузка процессора, снятие зависшего процесса через kill, перезапуск
системных служб через systemctl. Всё это выполняется на самом хосте. node-agent исполняет
переданный список аргументов в пространстве имён хоста через nsenter -t 1 -m -u -i -n -- <argv> и
возвращает код возврата, вывод и длительность.

## Почему привилегированный

Под запускается с securityContext.privileged: true и hostPID: true, монтируется в пространство
имён процесса с PID 1 (самого хоста) и потому обладает правами рута на узле. Это осознанная цена
возможности реально чинить узел (освобождать диск, снимать процессы, перезапускать службы). Из-за
этого поверхность закрыта несколькими рамками безопасности сразу.

## API

Сервис слушает 0.0.0.0 на порту NODEAGENT_PORT (по умолчанию 9110). При hostNetwork под слушает
порт узла напрямую, поэтому панель обращается к нему по адресу пода или узла внутри кластера.

Эндпоинт GET /health возвращает JSON вида {"status":"ok","node":"<NODE_NAME>"} и служит для
readiness-пробы, токен не требуется.

Эндпоинт POST /run исполняет команду. В заголовке обязателен X-NodeAgent-Token, который сверяется
с секретом NODEAGENT_TOKEN за постоянное время (hmac.compare_digest); без совпадения возвращается
401, ничего не исполняется (fail-closed). Тело запроса это JSON вида
{"argv":["df","-h","/"],"timeout":30}. Поле argv обязано быть непустым списком строк, а timeout
числом в пределах от 1 до 600 секунд; некорректное тело ведёт к ответу 400. Ответ имеет вид
{"exit_code":int,"stdout":str,"stderr":str,"duration_ms":int,"node":str}. Вывод по каждому из
потоков stdout и stderr ограничивается 256 килобайтами, при обрезке добавляется пометка
«...[обрезано]». При превышении таймаута возвращается exit_code, равный 124, и соответствующая
пометка в stderr.

## Переменные окружения

Переменная NODE_NAME задаёт имя узла и приходит через downward API из spec.nodeName; она
возвращается в ответах и попадает в логи. Переменная NODEAGENT_TOKEN это общий секрет доступа,
тот же самый, что кладётся в секрет adminchat под именем NODEAGENT_TOKEN. Переменная
NODEAGENT_PORT задаёт порт прослушивания и по умолчанию равна 9110.

## Безопасность

Доступ закрыт общим секретом: без валидного заголовка X-NodeAgent-Token эндпоинт /run отвечает
401 и не исполняет ничего, и точно так же fail-closed ведёт себя сервис, если серверный секрет не
задан вовсе. Сервис доступен только внутрикластерно: у него нет Ingress и нет Service типа
LoadBalancer, наружу узел не публикуется. Ключевая гарантия против инъекции оболочки состоит в
том, что команда всегда передаётся как список аргументов (argv), а исполнение идёт через
subprocess.run без shell=True и без sh -c. Метасимволы оболочки внутри argv остаются
литеральными аргументами и не интерпретируются, поэтому инъекция оболочки невозможна
конструктивно. Каждое исполнение пишется в структурный JSON-лог (argv, exit_code, duration_ms),
но сам токен доступа в лог никогда не попадает.

## Логи

Логи выводятся в stdout в структурном JSON по канону проекта (ADR-0032): поля ts, level,
service, равное node-agent, и msg, плюс контекст исполнения. Alloy забирает их в Loki.

## Тесты

Тесты лежат в test_node_agent.py и не требуют pytest и сети. Запуск: python3 test_node_agent.py.
Они проверяют валидацию тела, корректную сборку команды nsenter без превращения argv в строку
оболочки, обрезку вывода и fail-closed отказ без токена; subprocess при этом мокается, реальный
nsenter не вызывается.
